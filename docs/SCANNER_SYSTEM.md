# 📡 Система Сканера

## Обзор

Сканер — это ключевая система для обнаружения объектов в космосе. Без сканера игрок не может обнаруживать врагов, аномалии, дружественные корабли и другие опасные объекты до входа в них.

---

## 📋 Компоненты системы

### 1. Модуль Сканера

**Модуль `scanner`** — основной компонент для сканирования.

| Уровень  | Потребление | Диапазон | Здоровье | Защита |
|---------|-------------|----------|----------|--------|
| 1 | -2 | 3 | 100 | 1 |
| 2 | -4 | 5 | 120 | 2 |
| 3 | -6 | 8 | 140 | 3 |
| 4 | -9 | 15 | 200 | 4 |

**Цвет:** Голубой (`#00d4ff`)

**Важно:** Сканер должен быть **активен** (здоровье > 0, не отключён вручную) для работы.

### 2. Артефакты

#### 🔮 Квантовый сканер (Quantum Scan)
- **Эффект:** +5 к диапазону сканирования
- **Требование:** Должен быть установлен хотя бы один модуль сканера
- **Тип:** `quantum_scan`

#### 👁️ Глаз Сингулярности (All Seeing / Eye of Singularity)
- **Эффект:** Заменяет сканер, даёт диапазон 8 (эквивалент сканера 3 уровня)
- **Особенность:** Работает без модуля сканера
- **Тип:** `all_seeing`

### 3. Расовые бонусы

#### Кристаллический резонанс (Crystalline Resonance)
- **Эффект:** +15% к эффектам артефактов
- **Применение:** Увеличивает бонус от `quantum_scan`
- **Расовая принадлежность:** Требуется член экипажа с этой чертой

## 🎯 Пороги видимости объектов

Сканер обнаруживает объекты в зависимости от своего **эффективного диапазона**:

| Диапазон | Типы объектов |
|----------|---------------|
| **≥ 3** | Дружественные корабли (friendly_ship), Аномалии 1 тира |
| **≥ 5** | Аномалии 2 тира, Штормы (storm), Враги 2 тира |
| **≥ 8** | Враги 3 тира, Боссы (boss), Аномалии 3 тира |
| **≥ 15** | Аномалии 4 тира |

### Всегда видимые объекты
Следующие объекты видны **без сканера**:
- 🏠 Станции (station)
- 🪐 Планеты (planet)
- ☄️ Пояса астероидов (asteroid_belt)
- 📡 Сигналы бедствия (distress_signal) — но тип раскрывается сканером

---

## 🧮 Вычисление эффективного диапазона

### Формула

```typescript
эффективныйДиапазон = базовыйДиапазон + квантовыйБонус + технологическиеБонусы + резонанс
```

### Пошаговый расчёт

1. **Базовый диапазон** — максимальный `scanRange` среди активных модулей сканера
   - Если нет активных сканеров и нет артефакта "Глаз Сингулярности" → `0`
   - Если есть "Глаз Сингулярности" → `8`

2. **Бонус квантового сканера** — +5 если активен артефакт `quantum_scan`
   ```typescript
   if (quantumScanner) maxRange += 5
   ```

3. **Технологический бонус** — суммируются все изученные технологии с `scan_range`
   - Источники: технологии "Модуль сканера +1", "+2", "+3"
   - Формула: `maxRange += bonus1 + bonus2 + ...`
   - Пример: +1 +2 +3 = `maxRange + 6`

4. **Бонус кристаллического резонанса** — бонус только к артефакту `quantum_scan`
   - Требует: члена экипажа с чертой `resonance`
   - Бонус: +15% к эффектам артефактов
   - Применяется только к бонусу от `quantum_scan` (+5)
   - Формула: `maxRange += floor(5 × 0.15)`

### Пример расчёта

**Конфигурация:**
- Сканер 3 уровня (диапазон 8)
- Квантовый сканер (артефакт +5)
- Все 3 технологии (+1, +2, +3 = +6)
- Экипаж с кристаллическим резонансом

**Расчёт:**
1. База: `8`
2. Квантовый бонус: `8 + 5 = 13`
3. Технологический бонус: `13 + 6 = 19`
4. Резонанс (только к артефакту): `5 × 0.15 = 0.75 → 0` → `19 + 0 = 19`

**Итоговый диапазон:** `19`

## 🎮 Механика обнаружения

### При входе в локацию

Когда игрок входит в локацию, игра проверяет возможность сканирования:

```typescript
// Псевдокод из store.ts
const canScanEnemy = canScanObject("enemy", enemyTier);

if (!canScanEnemy && !loc.signalRevealed) {
    // Попытка раннего предупреждения
    const detected = Math.random() * 100 < earlyWarningChance;
    
    if (detected) {
        startCombat(loc); // Игрок готов к бою
    } else {
        gameMode = "unknown_ship"; // Неизвестный корабль
    }
} else {
    startCombat(loc); // Прямой бой
}
```

### Типы объектов и реакция

| Тип объекта | Проверка сканера | При неудаче |
|-------------|------------------|-------------|
| 🚁 **enemy** | `canScanObject("enemy", tier)` | Шанс раннего предупреждения → unknown_ship |
| 👹 **boss** | `canScanObject("boss", 3)` | Шанс раннего предупреждения → unknown_ship |
| 🌀 **anomaly** | `canScanObject("anomaly", tier)` | unknown_ship |
| 🤝 **friendly_ship** | `canScanObject("friendly_ship")` | unknown_ship |
| 🌩️ **storm** | `canScanObject("storm")` | Вход без предупреждения |
| 📡 **distress_signal** | Раскрытие типа | Тип остаётся скрытым |

## ⚔️ Раннее предупреждение

### Обнаружение засады

Если сканер не может обнаружить объект, есть шанс **раннего предупреждения**:

| Угроза | Порог | Базовый шанс | Бонус за пункт диапазона | Максимум |
|--------|-------|--------------|-------------------------|----------|
| **Тир 1** | 3 | 10% | +3% | 80% |
| **Тир 2** | 5 | 8% | +2.5% | 80% |
| **Тир 3 / Босс** | 8 | 5% | +2% | 60-80% |

**Формула:**
```typescript
шанс = базовыйШанс + (диапазон - порог) × бонусЗаПункт
шанс = min(максимум, шанс)
```

**Пример для врага 1 тира с диапазоном 7:**
- База: 10%
- Бонус: (7 - 3) × 3% = 12%
- Итого: 10% + 12% = **22%**

### Обнаружение босса

Для боссов используется отдельная формула в `store.ts`:
- База: 5%
- Порог: 8
- Бонус: +2% за пункт выше 8
- Максимум: 60%

**Пример с диапазоном 12:**
- База: 5%
- Бонус: (12 - 8) × 2% = 8%
- Итого: 5% + 8% = **13%**

## 📡 Раскрытие типа сигнала бедствия

### Механика

При первом входе в локацию с сигналом бедствия происходит проверка:

```typescript
const revealChance = getSignalRevealChance();
const canReveal = Math.random() * 100 < revealChance;

if (canReveal) {
    // Определяется тип сигнала (ambush, trap, etc.)
    loc.signalType = determineSignalType();
    loc.signalRevealed = true;
}
```

### Шансы раскрытия

| Диапазон | Базовый шанс | Бонус за пункт | Максимум |
|----------|--------------|----------------|----------|
| **≥ 15** | 75% | +2% | 95% |
| **≥ 8** | 50% | +2% | 95% |
| **≥ 5** | 30% | +2% | 95% |
| **≥ 3** | 15% | +2% | 95% |
| **< 3** | 0% | — | — |

**Пример с диапазоном 10:**
- База (≥ 8): 50%
- Бонус: (10 - 8) × 2% = 4%
- Итого: 50% + 4% = **54%**

## 🔬 Влияние технологий

### Технологии с бонусом к сканированию

| Технология | Уровень | Бонус | Стоимость |
|------------|---------|-------|-----------|
| 📡 **Модуль сканера +1** | Tier 1 | +1 к диапазону | 250₢ + 3 древ.данных |
| 🌀 **Модуль сканера +2** | Tier 2 | +2 к диапазону | 600₢ + 8 древ.данных + 1 кристалл |
| 🔮 **Модуль сканера +3** | Tier 3 | +3 к диапазону | 700₢ + 15 древ.данных + 2 кристалла |

**Применение:**
- Бонусы технологий **суммируются** (аддитивно)
- Формула: `диапазон + бонус1 + бонус2 + ...`
- Пример: все 3 технологии = +1 +2 +3 = **+6 к диапазону**

## 🎯 Стратегия использования

### Ранняя игра
1. Установите **сканер 1-2 уровня** для базового обнаружения
2. Изучите **Модуль сканера +1** для +1 к диапазону
3. Фокусируйтесь на обнаружении врагов 1-2 тира

### Средняя игра
1. Модернизируйте до **сканера 3 уровня** (диапазон 8)
2. Получите артефакт **Квантовый сканер** (+5 к диапазону)
3. Изучите **Модуль сканера +2** для +2 к диапазону

### Поздняя игра
1. Установите **сканер 4 уровня** (диапазон 15)
2. Получите **Глаз Сингулярности** для резервного сканирования
3. Изучите **Модуль сканера +3** для +3 к диапазону
4. Максимальный диапазон: **28+** со всеми бонусами
